Пользователь завершил задачу и хочет ревью. Это НЕ попытка решить задачу — решение уже написано.

## Что делать

1. Прочитай файл решения (пользователь укажет или ты видишь из контекста).
2. Запусти тесты: `go test ./... -v -run Test_<funcName>` чтобы проверить корректность.
3. Проанализируй и дай обратную связь по трём осям:

### Корректность
- Все ли edge cases покрыты? (nil, пустой ввод, один элемент, отрицательные числа)
- Есть ли логические ошибки?

### Сложность
- Какая временная сложность? Можно ли лучше?
- Какая пространственная? Есть ли лишняя память?

### Стиль Go
- Идиоматично ли написано для Go?
- Нет ли лишних аллокаций, ненужных переменных?

## Формат ответа

Три секции: **Корректность**, **Сложность**, **Стиль**. Коротко и по делу.
Если что-то можно улучшить — объясни *почему*, не переписывай код сам.
